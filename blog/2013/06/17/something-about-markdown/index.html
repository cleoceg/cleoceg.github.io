The purpose of Markdown

June 17, 2013

Markdown is an easy to read and write language which emulate a sort of HTML-light-tags. You can find a more precise description of it (Don’t Repeat Yourself – DRY) at wikipedia.

Already nine years old and not even an existing and approved RFC or draft of standard. It’s have already great support from lively projects like GitHub and Stack Overflow. You can see a good post on the subject at coding Horror where Jeff Atwood is proposing to bring forth a formal standard. For now I don’t know the status of this project/idea.

For my point of view I use Markdown with vim as a plugin. I find an entry point at vim.org and rely on the post of PlasticBoy to put it in my editor-sidekick: VIM. Although I’ve tried a more featured version of a dedicated editor for Markdown MarkdownPad. You can use the latter to view what your write in markdown tags and how it’ll be displayed in an HTML page. It is shipped with a free and a pro (but you must paid for it) version.

And last but not least this website DaringFireball give the official outlines of markdown “standard”

In Fact the only important question is: when does Markdown become a formal standard?
